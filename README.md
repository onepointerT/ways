# Ways

### The graphics compositor for wayland and distributed systems

##### v.0.0.0.dev0-pre1

The Ways project submits a compositor for wayland in userspace and real 3D graphics with HTML/CGI/CSS extensible markup for backend Xrandr/Cairo++/NK++. Every user space is implemented with `TypeScript`/`Ruby` and can be used via the `JavaScript` engine in the `HTML`/`CGI` on screen GUI gotten with `RubyCGI` and presented by `ways.wlcore`. Nothing of the all-over graphic engines will ever compose or access you users devices or folders, except `~/.wl/user` in a service-bus and in-memory manner and Write on service-thread-closed (WoSTC) with a localhost service-bus socket backend basing on `GoLang`/`gRPC`. The platinum compatibility and wsys/kernel independency will be made by using a HID (HumanInterfaceDevice: keyboard/mouse/screen) based on `ObjectiveC`-drivers for and with `Fortran#` and `C` compilation with `distcc` and `llvm` and a DCM-API (DriverContentManagement-API) for `wlcore`. Also it is now legal and nor distributes to have different device operating systems for the same screen/picture-sharing or vizualization with RestFul for screen-devices all distributed or embedded system understand a tensor picture format and use their own computer-local `WaysForWayLand` compositor or get it by an apparmor-secured TLSonVPN. This reduces -- compared to VNC-sharing -- CPU and warmness of the host system and optimizes, how and what a computer renders for `WayLand` basing on its own capacity strength.
Basing on whatever `Objective-C` compatible kernel, that means the only requirements for a system kernel are:

* Fortran-compatible with `C` and `F#`
* kmod/dkms for vnc/wlgo-sockets for localhost
* SysV-compatible and filesystem abstraction layer
* Alternative: LLVM/CLang-Compiler compatible

If it does not comply, a system developer would need a VNC-on-localhost service to make the wayland compositor and userspace markup grammar and class renderer compatible for the own system.

On more information on techniques and library features you could have a look at the `README.md` of the first two top-level directories.
